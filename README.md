# Reproducibility artifact for nonnormal measurement blocks

This repository contains the minimal code and generated numerical artifacts for the manuscript:

**Unitary Dilation Based Coherent Realization of Nonnormal Measurement Blocks in Quantum Weighted State Algorithms**


## Repository contents

- `scripts/generate_figures.py`: regenerates the operator-level computational figures used in the current manuscript: `Figure_1_Variance_Bound` and `Figure_3_Breakeven`.
- `scripts/gate_level_benchmark.py`: regenerates the compiled benchmark artifacts: `Figure_4_GateLevel_EndToEnd`, `Figure_5_GateLevel_Sensitivity`, `Figure_6_Ensemble_PhaseDiagram`, `Figure_7_Compiled_Predictors`, `Figure_8_Search_Workflow`, and the four CSV files in `data/`.
- `data/gate_level_resource_summary.csv`: per-shot compiled resources and ideal/noisy statistics for the two named operator families and three realization routes.
- `data/gate_level_sensitivity_summary.csv`: family sensitivity sweep over four workload depths and eight two-qubit noise values.
- `data/operator_ensemble_summary.csv`: 24-member random ensemble evaluated over four workload depths and eight two-qubit noise values, giving 768 compiled instances.
- `data/budgeted_search_summary.csv`: budgeted two-parameter search results for three realization routes and seven CNOT budgets.
- `figures/*.pdf`: final figure PDFs used in the current revised manuscript. `Figure_0_Overview.pdf` is a conceptual schematic with no underlying numerical data table; the computational figures are regenerated by the scripts above.
- `requirements.txt`: pinned Python package versions used for the artifact.

## Reproducing the artifacts

Create a clean Python environment and install the pinned dependencies:

```bash
python -m venv .venv
# Windows PowerShell:
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Regenerate the operator-level computational figures:

```bash
python scripts/generate_figures.py
```

Regenerate the gate-level benchmark figures and CSV tables:

```bash
python scripts/gate_level_benchmark.py
```

The gate-level script runs Qiskit transpilation and TensorCircuit/JAX simulations; it is expected to be slower than the operator-level figure script. Running either script overwrites the corresponding files in `figures/` and `data/`.

## Reproducibility parameters

The manuscript-relevant deterministic settings are encoded in the scripts:

- Random survey for `Figure_1_Variance_Bound`: 5000 complex 2 by 2 matrices, `seed=7`.
- Compiled ensemble: 24 complex 2 by 2 matrices, spectral norm 2, `ENSEMBLE_SEED=17`.
- Sensitivity grid: workload depths `1, 2, 3, 4` and two-qubit noise values from `2.5e-4` to `5.0e-3`.
- Budgeted search: seven CNOT budgets, 96 repeated trials per budget, `SEARCH_SEED=23`.
- Transpilation target: Qiskit optimization level 3 to the native `{u, cx}` basis.
